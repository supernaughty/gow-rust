From reqwest blocking (RESEARCH.md Pattern 6, verified docs.rs/reqwest/0.13.3):
```rust
use reqwest::blocking::{Client, ClientBuilder, Response};
use reqwest::Proxy;

// Plain GET
let client = ClientBuilder::new().build()?;
let response = client.get(&url).send()?;
let status = response.status();

// With proxy (Proxy::all matches curl -x behavior — proxies ALL protocols)
let proxy = Proxy::all("http://user:pass@proxy:8080")?;
let client = ClientBuilder::new().proxy(proxy).build()?;

// Write to file
let mut response = client.get(&url).send()?;
let mut file = File::create(&output_path)?;
io::copy(&mut response, &mut file)?;

// Write to stdout (bytes)
let bytes = response.bytes()?;
io::stdout().write_all(&bytes)?;

// HEAD request
let response = client.head(&url).send()?;
for (name, value) in response.headers() {
    println!("{}: {}", name, value.to_str().unwrap_or(""));
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied URL → HTTP request | URL is entirely user-controlled; SSRF is by design (curl is a user-controlled client) |
| HTTP response body → stdout/file | Server response is untrusted; written to stdout or user-specified file |
| TLS layer | Certificate validation by Windows SChannel against OS trust store |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06-01 | Spoofing | TLS certificate validation | mitigate | Never call `.danger_accept_invalid_certs(true)` unless `-k`/`--insecure` is explicitly passed by the user. Default: SChannel validates certs against Windows trust store. Document -k as a dangerous flag. |
| T-06-06-02 | Information Disclosure | HTTP redirect to file:// or other non-HTTP protocol | accept | reqwest 0.13 does not follow non-HTTP/HTTPS redirect targets by default. File:// redirects are rejected. Acceptable for a user-controlled CLI tool. |
| T-06-06-03 | Denial of Service | Response body larger than available disk/memory | accept | io::copy streams response body to file or stdout; no full buffer allocation. OS disk/pipe limits apply. User controls the URL. |
| T-06-06-04 | Tampering | Proxy credential exposure in process args | accept | `-x user:pass@proxy` passes credentials in CLI args (visible in ps output). This matches curl behavior and is expected for a CLI tool. No mitigation — document as known. |
| T-06-06-05 | Spoofing | Man-in-the-middle on HTTP (not HTTPS) | accept | HTTP is inherently unencrypted; HTTPS is the correct mitigation (user's responsibility). CLI curl does not force HTTPS. |
</threat_model>
