Replace crates/gow-gzip/src/lib.rs with full implementation:

1. Define `Cli` struct (clap derive) with these flags:
   - `-d` / `--decompress` (ArgAction::SetTrue): force decompress mode
   - `-c` / `--stdout` (ArgAction::SetTrue): write to stdout, keep original
   - `-k` / `--keep` (ArgAction::SetTrue): keep original files after compress/decompress
   - `files: Vec<String>`: positional operands (zero or more)

2. `Mode` enum: `Compress`, `Decompress`

3. `detect_mode(invoked_as: &str, decompress_flag: bool) -> Mode`:
   - Returns Decompress if invoked_as is "gunzip" or "zcat", or decompress_flag is true
   - Otherwise returns Compress

4. `is_stdout_mode(invoked_as: &str, stdout_flag: bool) -> bool`:
   - Returns true if invoked_as is "zcat" or stdout_flag is true

5. `compress_stream<R: Read, W: Write>(input: R, output: W) -> Result<()>`:
   - GzEncoder::new(output, Compression::default()), io::copy, encoder.finish()

6. `decompress_stream<R: Read, W: Write>(input: R, output: W) -> Result<()>`:
   - MultiGzDecoder::new(input), io::copy to output

7. `run(cli: Cli, invoked_as: &str) -> Result<i32>`:
   - Detect mode and stdout_mode from invoked_as + cli flags
   - If cli.files is empty: read stdin, compress or decompress to stdout
   - For each file path: convert via try_convert_msys_path, open, apply transform
   - Compress to file: open File::create(path + ".gz"), compress_stream, if !cli.keep: remove original
   - Decompress to file: strip ".gz" suffix for output name, decompress_stream, if !cli.keep: remove .gz
   - If stdout_mode: all output goes to io::stdout() regardless of compress/decompress
   - Per-file errors: eprintln!("gzip: {path}: {e}"), accumulate exit_code = 1, continue loop

8. `uumain<I: IntoIterator<Item = OsString>>(args: I) -> i32`:
   - gow_core::init() first
   - Collect args_vec, detect invoked_as from argv[0] file_stem to_lowercase
   - parse_gnu(Cli::command(), args_vec), Cli::from_arg_matches
   - call run(cli, &invoked_as), return result

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI args → file path operations | User supplies file paths; gzip reads and writes those paths |
| compressed stream → decompressor | Decompressed data comes from a potentially malicious .gz file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | gzip output file path derivation | mitigate | Output path = input + ".gz" or strip ".gz" — no path traversal possible; do not accept output path from archive metadata |
| T-06-02-02 | Denial of Service | malformed gzip bomb (compressed bomb decompresses to huge output) | accept | MultiGzDecoder streams to disk/stdout; no full-memory buffer; OS disk limits apply; gzip is not a server |
| T-06-02-03 | Tampering | overwrite existing files without warning | accept | GNU gzip overwrites by default (no -i prompt); matches GNU behavior; document as intentional |
| T-06-02-04 | Information Disclosure | stdin passthrough in decompress mode | mitigate | Guard per Pitfall 4: attempt GzDecoder; on decode error emit "not in gzip format" and exit 1 — never silently pass raw bytes through |
</threat_model>
