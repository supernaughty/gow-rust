<objective>
Create `.github/workflows/ci.yml` to run `cargo test --workspace` on every push to any branch and every pull request, using a windows-latest runner with Rust stable toolchain and dependency caching.

Purpose: Catch regressions automatically on every code change. All gow-rust utilities use Windows-specific APIs (windows-sys, UTF-8 console, Windows path handling) that require a Windows runner — Linux CI is not viable.

Output: `.github/workflows/ci.yml` — a GitHub Actions workflow file that triggers tests on push/PR and reports status to GitHub.
</objective>

<task type="auto">
  <name>Task 1: Create .github/workflows/ci.yml</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create the directory `.github/workflows/` if it does not exist, then write `.github/workflows/ci.yml` with the following exact content:

```yaml
name: CI
on:
  push:
    branches: ["**"]
  pull_request:
    branches: ["**"]

jobs:
  test:
    name: cargo test
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2
        with:
          key: x86_64-pc-windows-msvc

      - name: Run workspace tests
        run: cargo test --workspace
```

Key design decisions (per 07-RESEARCH.md):
- `branches: ["**"]` on push EXCLUDES tag pushes — prevents duplicate CI runs when v* tags are pushed (Pitfall 7 in research)
- `windows-latest` runner is mandatory — gow-rust utilities use Windows-only APIs (windows-sys, SetConsoleOutputCP, GetFileAttributesW) that cannot compile on Linux
- `dtolnay/rust-toolchain@stable` — NOT `actions-rs/toolchain` (archived/deprecated since 2023)
- `Swatinem/rust-cache@v2` with `key: x86_64-pc-windows-msvc` — differentiates this job's cache from the x86-release and x64-release keys used in release.yml
- No `permissions` block needed — CI only reads the repo (default read token is sufficient)
- No `--release` flag — debug builds are faster; tests do not require optimized binaries

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub PR author → CI runner | Untrusted code runs cargo test; no secrets available to PR builds |
| CI runner → GitHub status API | Reports pass/fail; uses GITHUB_TOKEN (read-only for CI job) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | ci.yml workflow definition | accept | Only maintainers with push access to master can modify workflow files; PRs from forks cannot modify workflow files |
| T-07-02-02 | Information Disclosure | Swatinem/rust-cache | accept | Cache contains compiled Rust artifacts only; no secrets; cache is keyed per branch and target |
| T-07-02-03 | Elevation of Privilege | GITHUB_TOKEN in CI job | accept | No `permissions: contents: write` on CI job; default read-only token cannot create releases or write to repo |
| T-07-02-04 | Tampering | Supply chain: dtolnay/rust-toolchain@stable | mitigate | Pin to `@stable` tag (not `@master`); tag is maintained by dtolnay; re-evaluate SHA pinning if supply chain audit is required in future |
</threat_model>
