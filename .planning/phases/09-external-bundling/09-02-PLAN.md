<objective>
Split the WiX installer harvest into two component groups — CoreComponents (Rust binaries only) and ExtrasComponents (vim, wget, nano, rg, batch shims) — and expose a user-deselectable "Extras" feature in the installer using WixUI_FeatureTree.

Purpose: BND-04 requires users to be able to opt out of vim/wget/nano during installation. Currently all files are harvested into a single BinComponents group with no way to deselect parts. After this plan, the installer shows a feature tree where "GOW-Rust Extras" can be unchecked.

Output: Modified build.bat (dual-subdirectory staging, dual heat.exe runs, updated candle/light commands) and modified wix/main.wxs (CoreComponents + ExtrasFeature, WixUI_FeatureTree).
</objective>

2. Replace the [2/4] staging step. The current step copies all Rust binaries and extras into the flat `%_STAGE%` directory. Replace it with separate staging into `core\` and `extras\`:

   ```batch
   echo [2/5] Staging Rust binaries to %_CORE_STAGE%
   powershell -NoProfile -Command "Remove-Item -Path '%_STAGE%' -Recurse -Force -ErrorAction SilentlyContinue; $null=New-Item '%_CORE_STAGE%' -ItemType Directory -Force; $null=New-Item '%_EXTRAS_STAGE%' -ItemType Directory -Force; Get-ChildItem 'target\%_RT%\release\*.exe' | Where-Object { $_.Name -ne 'gow-probe.exe' } | Copy-Item -Destination '%_CORE_STAGE%'"

   echo [3/5] Staging extras (vim, wget, nano, batch aliases) to %_EXTRAS_STAGE%
   if exist extras\bin (
       powershell -NoProfile -Command "Get-ChildItem 'extras\bin\*' -Include '*.exe','*.bat' | Copy-Item -Destination '%_EXTRAS_STAGE%'"
       if exist extras\bin\vim-runtime (
           powershell -NoProfile -Command "Copy-Item -Path 'extras\bin\vim-runtime' -Destination '%_EXTRAS_STAGE%\vim-runtime' -Recurse -Force"
       )
   )
   echo   Core staged:
   dir /b %_CORE_STAGE% 2>nul
   echo   Extras staged:
   dir /b %_EXTRAS_STAGE% 2>nul | findstr /v "^vim-runtime$"
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| extras/bin/ -> wix-stage/extras/ | Third-party binaries (vim, wget, nano, rg) are copied from extras/bin into the installer stage; their authenticity depends on download-extras.ps1 verification |
| Installer feature deselection | User can deselect ExtrasFeature; core Rust binaries remain mandatory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-02-01 | Tampering | extras/bin/*.exe (vim, wget, nano, rg) | accept | These are downloaded binaries; download-extras.ps1 uses TLS 1.2 and downloads from official release URLs (GitHub, eternallybored.org). SHA verification is out of scope for this phase — tracked as a future enhancement. |
| T-09-02-02 | Elevation of Privilege | MSI InstallScope=perMachine | accept | perMachine install requires UAC elevation — this is the correct and expected behavior for a system PATH registration tool. Users who run the installer have already accepted UAC. |
| T-09-02-03 | Denial of Service | heat.exe running on empty extras/ | mitigate | build.bat only copies extras if `extras\bin` exists. If the directory is empty, heat.exe on an empty dir produces a valid empty ComponentGroup — no crash. |
</threat_model>
