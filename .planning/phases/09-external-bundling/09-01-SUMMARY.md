# Phase 09 Plan 01: Batch Alias Shims — Source Control Commit Summary

**One-liner:** Eight legacy GOW batch alias shims (egrep, fgrep, bunzip2, gawk, gfind, gsort, gzip, unxz) committed to git so fresh clones provide compatibility names without running download-extras.ps1.

## What Was Done

The eight `.bat` alias shims that delegate to Rust binaries via `%~dp0`-relative paths were added to git source control. These files existed locally as untracked artifacts generated by `download-extras.ps1`, but were not committed. Committing them ensures:

1. **Fresh git clones** include all legacy GOW compatibility names immediately
2. **CI runners** do not need to run `download-extras.ps1` just to get the shims (the `.exe` binaries are still fetched separately)
3. **WiX installer stage** can copy shims from source without depending on the download script
