From RESEARCH.md (Pattern 4 — od output format engine):

GNU od default behavior with NO -t flag is `-t o2` (two-byte octal big-endian-style):
- Input: "abcdefgh\n" (9 bytes)
- Output:
  ```
  0000000 061141 062143 062145 064147 005150
  0000011
  ```
- Address column: 7-digit octal, padded with leading zeros, separated by single space from data
- Final line: a single address representing total bytes consumed, no data

Type letters and unit sizes:

| -t | Meaning | Output Format |
|----|---------|---------------|
| `o` or `o2` | 2-byte octal | %06o per 16-bit word, little-endian on x86 (so 0x6261 -> octal 061141 for "ab") |
| `o1` | 1-byte octal | %03o per byte |
| `x` or `x2` | 2-byte hex | %04x per 16-bit word |
| `x1` | 1-byte hex | %02x per byte |
| `x4` | 4-byte hex | %08x per 32-bit word |
| `d2` | 2-byte signed decimal | space-padded |
| `d4` | 4-byte signed decimal | space-padded |
| `u1` | 1-byte unsigned decimal | %3u |
| `u2` | 2-byte unsigned decimal | %5u |
| `c` | char per byte | printable ASCII as char, control chars as named (\\n, \\t) or octal |

Address letters:

| -A | Format | Example for offset 16 |
|----|--------|----------------------|
| `o` (default) | 7-digit octal | `0000020` |
| `x` | 7-digit hex | `0000010` |
| `d` | 7-digit decimal | `0000016` |
| `n` | suppress address | (empty prefix) |

Endianness note: x86 is little-endian. For -t o2 on bytes [0x61, 0x62] ("ab"), the 16-bit word read as little-endian is 0x6261 = octal 061141. Use `u16::from_le_bytes([b0, b1])` to read.

GNU od line behavior:
- 16 bytes per row regardless of unit size
- For -t o2: 8 words per row, each word printed with single-space separator
- For -t o1: 16 values per row
- For -t c: 16 chars per row, each in 4-char field (`%4s`) — printable as `   a`, control as `  \\n` etc.

Empty input: still emit "0000000\n"

Format helpers:

```rust
fn format_address(offset: u64, fmt: AddrFmt) -> String {
    match fmt {
        AddrFmt::Octal   => format!("{:07o}", offset),
        AddrFmt::Hex     => format!("{:07x}", offset),
        AddrFmt::Decimal => format!("{:07}", offset),
        AddrFmt::None    => String::new(),
    }
}

fn format_value(bytes: &[u8], spec: TypeSpec) -> String {
    use TypeSpec::*;
    match spec {
        Octal(1)        => format!("{:03o}", bytes[0]),
        Octal(2)        => format!("{:06o}", u16::from_le_bytes([bytes[0], bytes[1]])),
        Octal(4)        => format!("{:011o}", u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]])),
        Hex(1)          => format!("{:02x}", bytes[0]),
        Hex(2)          => format!("{:04x}", u16::from_le_bytes([bytes[0], bytes[1]])),
        Hex(4)          => format!("{:08x}", u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]])),
        SignedDec(1)    => format!("{:4}", bytes[0] as i8),
        SignedDec(2)    => format!("{:6}", i16::from_le_bytes([bytes[0], bytes[1]])),
        SignedDec(4)    => format!("{:11}", i32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]])),
        UnsignedDec(1)  => format!("{:3}", bytes[0]),
        UnsignedDec(2)  => format!("{:5}", u16::from_le_bytes([bytes[0], bytes[1]])),
        UnsignedDec(4)  => format!("{:10}", u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]])),
        Char            => format_char(bytes[0]),
        // 8-byte sizes for octal/hex/dec follow same pattern; include if straightforward.
        _               => format!("{:?}", bytes),
    }
}

fn format_char(b: u8) -> String {
    match b {
        0x07 => "  \\a".to_string(),
        0x08 => "  \\b".to_string(),
        0x09 => "  \\t".to_string(),
        0x0A => "  \\n".to_string(),
        0x0B => "  \\v".to_string(),
        0x0C => "  \\f".to_string(),
        0x0D => "  \\r".to_string(),
        0x00 => "  \\0".to_string(),
        b if (0x20..=0x7E).contains(&b) => format!("{:>4}", b as char),
        b => format!("{:>4}", format!("{:03o}", b)),
    }
}
```

run(cli) logic:
1. Parse cli.address into AddrFmt (on error: eprintln "od: invalid output address radix '<x>'", return 1)
2. Parse cli.type_spec into TypeSpec (on error: eprintln "od: invalid type string '<x>'", return 1)
3. Determine `unit_size` from TypeSpec (1 for Char, else size)
4. Determine `bytes_per_line = 16` (GNU default)
5. Determine `values_per_line = bytes_per_line / unit_size`
6. Open input(s):
   - If files.is_empty(): use stdin
   - Else: chain files together; on file open error eprintln "od: <path>: <err>", set exit_code=1, continue
7. Read up to read_bytes (or unlimited) into a buffer (or stream — for Phase 10, reading all input into memory is acceptable for typical use; document as limitation)
8. Iterate over the buffer in `bytes_per_line` chunks:
   - Print address (format_address(offset, addr_fmt))
   - For each `unit_size` chunk in the row, print " " + format_value(...)
   - Print "\n"
   - offset += chunk.len()
   - HANDLE PARTIAL LAST ROW: if remaining bytes < bytes_per_line, still emit the row with the values for the bytes actually present (no padding values for missing bytes — GNU od just stops mid-row)
   - Special case: if buffer is empty, still emit "0000000\n" trailer below
9. After all data: emit FINAL line with the total byte count formatted as an address (no data, no leading space): `format_address(total_bytes, addr_fmt) + "\n"`. If addr_fmt is None, emit empty line "\n".
