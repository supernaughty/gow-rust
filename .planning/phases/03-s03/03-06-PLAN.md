Implement `gow-unix2dos` (CONV-02): replace the stub from Plan 03-01 with an LF→CRLF in-place converter. This is a mirror of `gow-dos2unix` (Plan 03-05) that SHARES the byte-level transform module via a path dependency — no duplicated scanner code.

Why `depends_on: ["03-05"]`: the `transform.rs` scanner + the `pub mod transform` export are created in 03-05. We add a Cargo.toml path dependency `gow-dos2unix = { path = "../gow-dos2unix" }` and call `uu_dos2unix::transform::lf_to_crlf` directly. This matches the PATTERNS.md Pattern M recommendation (option b: "put both in one crate and cross-depend"). It avoids creating a third workspace member for ~30 lines of shared code.

Why wave 3 instead of wave 2: the cross-crate dependency on a Wave 2 crate forces sequential execution. Could run in wave 2 if planner chose to (no file conflicts with 03-05), but making it wave 3 avoids any race with 03-05's `pub mod transform` export landing. Trade-off: small serialization loss vs. guaranteed consumption order. Acceptable.

Output: `unix2dos.exe` real binary, 10+ integration tests including a round-trip test that calls BOTH `target/debug/dos2unix` and `target/debug/unix2dos` via assert_cmd to prove ROADMAP criterion 5 end-to-end.
