Phase: 03 (s03) — Milestone: M001

Workspace-level preparation for Phase 3 filesystem utilities. This plan is the single blocking dependency for every other Phase 3 plan (Waves 1-5). It does three things in parallel-safe file ownership:

1. Extends `crates/gow-core/src/fs.rs` with seven new helpers (atomic_rewrite, create_link, LinkKind/LinkOutcome, is_hidden, is_readonly, has_executable_extension, clear_readonly, is_drive_root). These encapsulate all Windows-specific link/attribute logic so utility crates remain cross-platform Rust.

2. Amends the workspace `Cargo.toml` to add four new workspace dependencies (walkdir 2.5, notify 8.2, terminal_size 0.4, junction 1.4) and registers 11 new crate members. Also promotes tempfile from dev-dep to runtime dep in gow-core (needed by atomic_rewrite).

3. Scaffolds 11 stub crates (gow-cat, gow-ls, gow-cp, gow-mv, gow-rm, gow-ln, gow-chmod, gow-head, gow-tail, gow-dos2unix, gow-unix2dos) using the Phase 2 gow-touch template verbatim. Each stub prints `{name}: not yet implemented` and exits 1 — same pattern as Phase 2 Plan 02-01 (STATE.md entry confirms 14 Phase 2 crates were scaffolded this way up-front).

Purpose: All 11 utility plans in Waves 1-5 can run in parallel worktrees once this plan lands. Without it, every later plan would need to edit the workspace root Cargo.toml and would conflict.

Output: Workspace compiles. 11 new `{name}.exe` stubs present in `target/debug/`. `cargo test --workspace` passes (existing tests only; stubs have one smoke test each). `cargo clippy --workspace --all-targets -- -D warnings` is clean.
