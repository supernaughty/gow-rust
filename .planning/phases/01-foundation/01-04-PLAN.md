Phase: 01 (foundation) — Milestone: M001

Create the `gow-probe` test binary and integration test suite. gow-probe is a minimal binary crate (not shipped to users) that calls `gow_core::init()` and exercises all six gow-core modules via its CLI arguments. The integration tests use assert_cmd to spawn gow-probe and verify observable behavior: exit codes, stdout output, and path conversion.

Purpose: Phase 1 produces no user-facing utilities. gow-probe provides a runnable binary to validate that gow-core's platform primitives work end-to-end, not just in unit tests. This is the only way to verify WIN-01, WIN-02, WIN-03 at the binary level.
Output: gow-probe binary crate + integration tests; `cargo test --workspace` green.
