| -r path argument → filesystem | Recursive walk starting from user-supplied directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Denial of Service | regex engine (ReDoS) | mitigate | Use `regex` crate (linear-time, no catastrophic backtracking). Backreferences and lookaheads are intentionally unsupported. Document this deviation from GNU grep -P behavior. |
| T-04-03-02 | Information Disclosure | recursive walk (-r) | mitigate | WalkDir follows filesystem permissions; errors on unreadable files are printed to stderr (not silenced). No path traversal beyond user-specified root. |
| T-04-03-03 | Elevation of Privilege | regex pattern from args | accept | grep does not execute code; patterns are only matched against input bytes. No shell interpolation. |
| T-04-03-04 | Denial of Service | large directory trees (-r) | accept | File handles are opened and closed per-file in WalkDir iteration; no handle exhaustion. Memory is O(line_length). |
</threat_model>
