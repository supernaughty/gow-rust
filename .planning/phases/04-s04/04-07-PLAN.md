       ```rust
       enum Expr {
           Num(f64), Str(String), Var(String),
           FieldAccess(Box<Expr>),   // $0, $1, $NF
           BinOp(Box<Expr>, BinOp, Box<Expr>),
           UnOp(UnOp, Box<Expr>),
           ArrayAccess(String, Box<Expr>), // arr[key]
           Concat(Box<Expr>, Box<Expr>),
           Regex(String),
           FnCall(String, Vec<Expr>),  // print, printf, length, substr, split, gsub, sub, match, toupper, tolower
           Getline,
       }
       enum BinOp { Add, Sub, Mul, Div, Mod, Pow, Eq, Ne, Lt, Le, Gt, Ge, And, Or, In, Append, Match, NotMatch }
       enum Stmt {
           Print(Vec<Expr>, Option<Expr>),   // print exprs [> file]
           Printf(Vec<Expr>, Option<Expr>),
           Assign(String, Expr),
           FieldAssign(Expr, Expr),
           ArrayAssign(String, Expr, Expr),
           If(Expr, Box<Stmt>, Option<Box<Stmt>>),
           While(Expr, Box<Stmt>),
           For(Option<Box<Stmt>>, Option<Expr>, Option<Box<Stmt>>, Box<Stmt>),
           ForIn(String, String, Box<Stmt>),
           Do(Box<Stmt>, Expr),
           Block(Vec<Stmt>),
           Delete(String, Option<Expr>),
           Next, Exit(Option<Expr>),
           Return(Option<Expr>),
           Expr(Expr),
       }
       struct Rule { pattern: Pattern, action: Vec<Stmt> }
       enum Pattern { Begin, End, Expr(Expr), Range(Expr, Expr), Always }
       struct Program { rules: Vec<Rule> }
       ```

    **Interpreter state:**
    ```rust
    struct Env {
        globals: HashMap<String, Value>,
        arrays: HashMap<String, HashMap<String, Value>>,
        fields: Vec<String>,   // $0 is fields[0], $1 is fields[1], etc.
        nr: usize,             // NR (record number across all files)
        fnr: usize,            // FNR (record number in current file)
        nf: usize,             // NF
        fs: String,            // FS field separator
        ofs: String,           // OFS output field separator
        rs: u8,                // RS record separator (default '\n')
        ors: String,           // ORS output record separator
        filename: String,      // FILENAME
    }
    ```

    **Value type:**
    ```rust
    enum Value { Num(f64), Str(String), Uninitialized }
    impl Value {
        fn to_num(&self) -> f64;   // numeric coercion
        fn to_str(&self) -> String; // string coercion
    }
    ```

    **Field splitting:** `fn split_fields(record: &str, fs: &str) -> Vec<String>`:
    - If FS is " " (default): split on runs of whitespace, trim leading/trailing
    - If FS is single char: split on that char
    - If FS is multi-char: treat as regex, split with Regex::new(fs)?

    **Expression evaluation:** `fn eval_expr(expr: &Expr, env: &mut Env) -> Result<Value>`:
    - Numeric ops: coerce both sides to f64
    - String concat: coerce both to string, concatenate
    - `$N`: return env.fields[N] or "" if out of range; `$0` = full record
    - Array access: `arr[key]` returns env.arrays["arr"]["key"] or Uninitialized
    - Regex match `$0 ~ /pat/`: compile pat, test against $0
    - Built-in functions:
      - `length(s)` → number of chars; `length(arr)` → number of keys
      - `substr(s, start[, len])` → substring (1-indexed)
      - `split(s, arr, sep)` → split s into arr[1..N], return N
      - `gsub(re, repl[, str])` → global substitution, return count
      - `sub(re, repl[, str])` → first substitution
      - `match(s, re)` → sets RSTART/RLENGTH, returns index or 0
      - `toupper(s)` / `tolower(s)`
      - `sprintf(fmt, ...)` → formatted string (use Rust format_printf logic below)
      - `int(n)` → truncate to integer

    **printf/sprintf formatting:** Implement `fn format_printf(fmt: &str, args: &[Value]) -> String`:
    - Support: `%s` (string), `%d` (integer), `%f` (float), `%e`/`%E` (scientific), `%g`/`%G`, `%x`/`%X` (hex), `%o` (octal), `%c` (char)
    - Width/precision: `%05d`, `%-10s`, `%.2f`
    - `%%` literal percent

    **Statement execution:** `fn exec_stmt(stmt: &Stmt, env: &mut Env) -> Result<ControlFlow>`:
    - Print: evaluate exprs, join with OFS, write to stdout + ORS
    - Printf: call format_printf, write result
    - Assign: update env.globals
    - ArrayAssign: update env.arrays
    - If/While/For/ForIn/Do: standard control flow
    - Next: return ControlFlow::Next (skip to next record)
    - Exit: return ControlFlow::Exit(code)

    **Main execution loop** in `fn execute(program: &Program, args: &Args) -> Result<i32>`:
    - Initialize Env from -v assignments and -F separator
    - Run all BEGIN rules (no input needed)
    - For each input file (or stdin if no files): read records split by RS (default '\n'), split fields with FS, run matching per-record rules
    - Pattern matching: evaluate pattern expr, compare to $0; regex patterns compiled once
    - Run all END rules
    - Return 0

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| awk program text → interpreter | User-supplied AWK program parsed and executed against input |
| stdin/file → awk record processing | Untrusted byte stream split into fields and processed |
| awk print output redirect (>) | AWK program can write to files via print > "file" |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07-01 | Denial of Service | awk regex in pattern/gsub (ReDoS) | mitigate | Use `regex` crate (linear-time guarantees). Document that backreferences are unsupported (deviation from gawk). |
| T-04-07-02 | Denial of Service | infinite while loop in awk program | accept | AWK is a scripting language; user-supplied programs may loop indefinitely. This is by design (same as GNU awk). No timeout imposed. |
| T-04-07-03 | Information Disclosure | awk print redirect (> "file") | mitigate | Only implement redirect if explicitly needed for R013. For initial implementation, disable `print > file` syntax (return error "redirect not supported") to prevent arbitrary file writes. |
| T-04-07-04 | Elevation of Privilege | awk system() / pipe calls | mitigate | Do NOT implement `system()`, `|` pipe, or `getline` from command. These execute shell commands. Omit entirely from this implementation — return runtime error "system() not supported". |
| T-04-07-05 | Denial of Service | associative array unbounded growth | accept | User-supplied data can fill associative arrays. This matches GNU awk behavior; bounded by system memory. |
| T-04-07-06 | Tampering | -v variable injection | mitigate | Parse -v as "VAR=VALUE" only (split on first '='). Reject variable names containing non-alphanumeric/underscore chars. |
</threat_model>
