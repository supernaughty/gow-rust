**Critical note on `true` / `false`:** Rust keywords `true` and `false` cannot be used as unquoted identifiers. In `Cargo.toml` the string values `"true"` and `"false"` are fine (TOML strings, not identifiers). In `src/lib.rs` / `src/main.rs` / `[lib] name = "uu_true"` these are fine because they become module names `uu_true` / `uu_false` (letter prefix), and TOML `name = "true"` is just a string. The binary executable will be named `true.exe` / `false.exe` on Windows — this is what we want (D-16d).
