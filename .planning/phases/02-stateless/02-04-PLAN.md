Phase: 02 (stateless) — Milestone: M001

Implement `gow-pwd` with `-L` (logical, default) and `-P` (physical) flags, plus a safe `\\?\` UNC-aware prefix strip function (dunce::simplified-equivalent) inlined in `src/canonical.rs`. This delivers UTIL-02.

Purpose: GNU `pwd` is trivial on Unix but on Windows requires UNC prefix handling because `std::fs::canonicalize` always prefixes paths with `\\?\`. Naive `strip_prefix(r"\\?\")` corrupts true-UNC `\\?\UNC\server\share\…` paths. RESEARCH.md Q8 documents the exact safety rule; this plan implements it directly.
Output: `pwd.exe` binary with -L/-P flag and UNC-safe canonicalization; 6-8 integration tests + 4-5 unit tests on `simplify_canonical`.
