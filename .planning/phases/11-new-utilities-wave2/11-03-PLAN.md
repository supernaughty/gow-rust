split suffix generator (from RESEARCH.md Pattern 6 — implement exactly this):
```rust
fn next_suffix(suffix: &mut Vec<u8>) {
    let mut i = suffix.len() - 1;
    loop {
        if suffix[i] < b'z' {
            suffix[i] += 1;
            return;
        }
        suffix[i] = b'a';
        if i == 0 {
            // All z's — extend: zz → aaa, zzz → aaaa, etc.
            suffix.iter_mut().for_each(|c| *c = b'a');
            suffix.push(b'a');
            return;
        }
        i -= 1;
    }
}
```

join merge algorithm pseudocode (from RESEARCH.md Pattern 6):
```
Read one record from each file into line1/line2 (Option<String>).
Loop while either line1 or line2 is Some:
  Extract join_field from each current record.
  Compare fields lexicographically:
    Equal → output joined line; advance BOTH files
    line1_field < line2_field → if -a 1, output line1; advance file1 only
    line1_field > line2_field → if -a 2, output line2; advance file2 only
  When one file exhausted: process remaining other file lines with -a rules.
```

```rust
}
```

run(cli) logic:

1. Validate: if suffix_len == 0 → eprintln "split: suffix length must be > 0", return 1

2. Parse byte size if -b given:
```rust
fn parse_bytes(s: &str) -> Option<usize> {
    let s = s.trim();
    let (num_str, mult) = if s.ends_with('G') || s.ends_with('g') {
        (&s[..s.len()-1], 1024*1024*1024usize)
    } else if s.ends_with('M') || s.ends_with('m') {
        (&s[..s.len()-1], 1024*1024)
    } else if s.ends_with('K') || s.ends_with('k') {
        (&s[..s.len()-1], 1024)
    } else {
        (s, 1)
    };
    num_str.parse::<usize>().ok().map(|n| n * mult)
}
```
If parse_bytes returns None → eprintln "split: invalid number of bytes: '<arg>'", return 1.
If result == 0 → same error.

3. Read input: if cli.input is None or == "-", read stdin to `Vec<u8>`. Else read file (apply try_convert_msys_path).

4. Determine prefix: cli.prefix.unwrap_or_else(|| "x".to_string())

5. Initialize suffix: `vec![b'a'; cli.suffix_len]`

6. Split mode dispatch:
   - **-l mode** (or default 1000 lines when no -b/-n/-l given):
     - line_count = cli.lines.unwrap_or(1000)
     - If line_count == 0 → eprintln "split: invalid number of lines: '0'", return 1
     - Iterate over lines in input buffer (split on `b'\n'`):
       - Accumulate `line_count` lines per output file
       - For each chunk, create file named `{prefix}{suffix_str}`, write chunk, call `next_suffix(&mut suffix)`
   - **-b mode**:
     - byte_count from parse_bytes
     - Chunk input Vec<u8> into slices of byte_count bytes
     - Write each chunk to `{prefix}{suffix_str}`, call `next_suffix`
   - **-n mode**:
     - chunks = cli.chunks. If 0 → error.
     - total = input.len(); chunk_size = (total + chunks - 1) / chunks (ceiling division)
     - Write each chunk (last chunk gets remainder), call `next_suffix`

7. Output file creation:
```rust
let suffix_str = String::from_utf8_lossy(&suffix).to_string();
```

run(cli) logic:

1. Validate files: must have exactly 2 entries. If not: eprintln "join: invalid usage: need exactly 2 files", return 2.
2. Open each file as `BufReader<Box<dyn Read>>` (stdin for "-", File::open for paths).
3. Parse separator: `let sep: Option<char> = cli.separator.as_ref().and_then(|s| s.chars().next());`
4. Field extraction function:
```rust
fn get_field<'a>(line: &'a str, field: usize, sep: Option<char>) -> &'a str {
    if let Some(c) = sep {
        line.splitn(field + 1, c).nth(field - 1).unwrap_or("")
    } else {
        line.split_whitespace().nth(field - 1).unwrap_or("")
    }
}
```
5. Implement merge-join loop:
   - Maintain `line1: Option<String>` and `line2: Option<String>`, initialized by reading the first line from each file.
   - Also maintain `prev_key1: Option<String>` and `prev_key2: Option<String>` for sort-order warning.
   - Loop:
     - If both None → break
     - If one is None: if -a or -v for the non-exhausted file, print remaining lines; break
     - Extract key1 from line1 (using field1, sep), key2 from line2 (using field2, sep)
     - Compare keys:
       - Equal: if only_unpaired is NOT set for both (i.e., normal join output): print joined line. Advance both.
       - key1 < key2: if print_unpaired == Some(1) or only_unpaired == Some(1): print line1. Advance file1 only.
       - key1 > key2: if print_unpaired == Some(2) or only_unpaired == Some(2): print line2. Advance file2 only.
     - Sort-order check: if key1 < prev_key1 (current key is less than previous key for same file), eprintln warning to stderr: "join: file 1 is not in sorted order"
   - Joined output format: `{key} {file1_other_fields} {file2_other_fields}` separated by spaces (or sep char if -t given).

"Other fields" = all fields except the join field. Reconstruct by joining remaining parts:
```rust
fn other_fields(line: &str, join_field: usize, sep: Option<char>) -> String {
    if let Some(c) = sep {
        let parts: Vec<&str> = line.splitn(usize::MAX, c).collect();
        parts.iter().enumerate()
            .filter(|(i, _)| i + 1 != join_field)
            .map(|(_, s)| *s)
            .collect::<Vec<_>>()
            .join(&c.to_string())
    } else {
        line.split_whitespace().enumerate()
            .filter(|(i, _)| i + 1 != join_field)
            .map(|(_, s)| s)
            .collect::<Vec<_>>()
            .join(" ")
    }
}
```

Output separator between join field and other fields: space (or sep char if -t given).
Print joined line: `writeln!(out, "{key} {f1_other} {f2_other}")` — if f1_other or f2_other is empty, skip the trailing space.
