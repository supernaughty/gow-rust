# Quick Task 260429-q01: Multi-Arch Build + WiX MSI Infrastructure Summary

**One-liner:** Added x86/ARM64 CRT-static build targets, WiX v3 MSI pipeline with heat-harvested binaries, and setup.bat toolchain installer to enable single-command multi-arch MSI generation.

## What Was Built

### Task 1: `.cargo/config.toml` — CRT Static Linking (commit `6e2a7c7`)

Extended the existing config with two new target sections:

- `i686-pc-windows-msvc` — x86 32-bit Windows
- `aarch64-pc-windows-msvc` — ARM64 Windows

All three MSVC targets now pass `-C target-feature=+crt-static`, embedding VCRUNTIME140.dll
into binaries so they run without the MSVC redistributable installed on the target machine.

### Task 2: `setup.bat` — Toolchain Prerequisites (commit `1da930f`)

One-shot setup script for a fresh Windows machine:

1. `rustup target add i686-pc-windows-msvc`
2. `rustup target add aarch64-pc-windows-msvc`
3. `cargo install cargo-wix --version 0.3.9`
4. `winget install WiXToolset.WiXToolset` (WiX v3.14.1)
5. Verifies `heat.exe` is on PATH; warns with add-to-PATH instructions if not

Prints a summary with next steps (`build.bat installer x64`).

### Task 3: `build.bat` — `--arch` Flag + `installer` Subcommand (commit `ca9f13b`)

New capabilities:

- `--arch x64|x86|arm64` flag maps to the corresponding MSVC target triple
- Works with both debug/release builds and `-p <crate>` scoping
- `installer [x64|x86|arm64|all]` subcommand runs the full WiX pipeline:
  1. `cargo build --release --target <triple>`
  2. Stage all `.exe` to `target\wix-stage\<arch>\`
  3. `heat.exe` harvest to `wix\BinHarvest-<arch>.wxs`
  4. `candle.exe` compile `main.wxs` + harvest fragment
  5. `light.exe` link to `target\gow-rust-<arch>.msi`
- `installer all` loops all three architectures sequentially
- Updated help section documents all new commands

### Task 4: `wix/main.wxs` + `wix/License.rtf` — MSI Product Definition (commit `e1d6fb1`)

WiX v3 product definition:

- **Product:** GOW-Rust 0.1.0, UpgradeCode `{A1B2C3D4-E5F6-7890-ABCD-EF1234567890}`
- **Install dir:** `C:\gow-rust` (default, user-configurable at install time)
- **PATH registration:** `<Environment>` element in `PathEnvComponent`, system-scope, non-permanent (removed on uninstall)
- **MajorUpgrade:** removes previous version before install
- **Architecture:** preprocessor conditionals on `$(var.Platform)` for x64/arm64 vs x86 (Win64 flag, ProgramFiles64Folder)
- **Binaries:** wired via `<ComponentGroupRef Id="BinComponents"/>` — populated by heat at build time
- **UI:** WixUI_Minimal with MIT License.rtf dialog

## Deviations from Plan

### Auto-added: `wix/License.rtf`

- **Found during:** Task 4
- **Issue:** WixUI_Minimal requires `WixUILicenseRtf` variable to point to a valid RTF file; without it, `light.exe` fails at link time. The plan referenced the WixUI_Minimal UI but did not include a license file.
- **Fix:** Created `wix/License.rtf` with the MIT license text in RTF format and included it in the same commit as `wix/main.wxs`.
- **Files modified:** `wix/License.rtf` (new), `wix/main.wxs` (WixVariable element)
- **Rule:** Rule 2 (missing critical functionality — build would fail without it)

- **Found during:** Task 1
- **Issue:** File already existed with `[build] target = x86_64-pc-windows-msvc` and the x64 CRT-static rustflag.
- **Fix:** Appended i686 and aarch64 sections; preserved existing content exactly.
